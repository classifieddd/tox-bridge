Tox Bridge

Un bot Telegram qui parle le protocole Tox.
Messages chiffrés de bout en bout sans serveur central.

@pursiveBot Code source

Présentation

Tox est un réseau pair-à-pair (P2P) sans serveur central. Les messages vont directement d’un appareil à l’autre, chiffrés de bout en bout. Tox Bridge fait le lien entre Telegram et Tox : vous utilisez Telegram (que vous avez déjà), le bot incarne votre identité Tox et transmet les messages.

Démarrage rapide

  1. /start — le bot crée votre profil Tox
  2. /myid — récupérez votre adresse à partager
  3. Envoyez cette adresse à votre contact (Signal, SMS, …)
  4. Votre contact fait /add <adresse> sur son bot
  5. Vous recevez une notification → /accept
  6. /select <numéro> pour choisir un contact
  7. Écrivez un message — le bot le transmet sur le réseau Tox

Comment ça marche

Architecture

Chaque utilisateur Telegram possède sa propre identité Tox (paire de clés). Elle est stockée dans un fichier .tox sur le serveur. Les messages transitsur le réseau Tox normal — ils sont chiffrés de bout en bout par la bibliothèque libtoxcore. Le bot ne fait que relayer le texte entre Telegram et Tox.

Réseau

Le bot utilise les nœuds d’amorçage publics de Tox (DHT) pour rejoindre le réseau. Une fois connecté, chaque identité est joignable via son adresse (clé publique + nospam). Le bot se connecte également à des relais TCP (ports 443, 3389, 33445) pour garantir la connectivité même derrière un NAT ou un pare-feu restrictif.

Sécurité des données

Chiffrement de la base

La base de données (bot.db) est chiffrée au repos avec Fernet (AES-128-CBC + HMAC-SHA256). La clé est dérivée via PBKDF2-SHA256 (200 000 itérations) avec un sel aléatoire de 16 octets préfixé au fichier. Sans la variable d’environnement TOX_DB_KEY, la base est illisible.

Profils Tox

Chaque profil (profile_{id}.tox) contient la clé privée, la liste d’amis et les paramètres. Écriture atomique (fichier temporaire + renommage) pour éviter toute corruption. Sauvegarde automatique toutes les 30 secondes et après chaque modification (ajout/suppression d’ami, changement de nom).

Ce qui n’est PAS sauvegardé

  • Les messages échangés (ils transitent uniquement en mémoire)
  • Les logs de conversation
  • Les métadonnées d’utilisation

Persistance

  • Identité Tox : sauvegardée dans le profil .tox
  • Liste d’amis : incluse dans le profil (natif libtoxcore)
  • Demandes d’ami en attente : base chiffrée
  • Contact sélectionné : base chiffrée
  • Pseudo des amis : base chiffrée
  • Backup automatique : quotidien à 4h UTC, conservation 7 jours

Déploiement

Pré-requis

  • Ubuntu 24.04+ (ou Debian 12+)
  • Python 3.12+
  • libtoxcore 0.2.22 (apt install libtoxcore-dev)
  • Un token bot Telegram
  • Une variable d’environnement TOX_DB_KEY (32+ caractères)

Installation

git clone https://github.com/classifieddd/tox-bridge.git
cd tox-bridge
python3 -m venv venv
. venv/bin/activate
pip install python-dotenv python-telegram-bot cryptography toxygen
# Éditer .env :
#   TELEGRAM_BOT_TOKEN=votre_token
#   TOX_DB_KEY=votre_cle_maitresse
python run.py

Systemd (production)

[Unit]
Description=Tox-Telegram Bridge
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/tox-bridge
EnvironmentFile=/home/ubuntu/tox-bridge/.env
ExecStart=/home/ubuntu/tox-bridge/venv/bin/python /home/ubuntu/tox-bridge/run.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

Dépendances

  • python-telegram-bot — interface Telegram
  • cryptography — chiffrement Fernet de la base
  • libtoxcore — pile protocolaire Tox (binaire système)
  • python-dotenv — chargement du fichier .env
